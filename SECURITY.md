# Security policy

Do not open public issues containing secrets, private evidence, undisclosed vulnerabilities, account data, or target-specific reproduction details.

The project accepts only authorized, non-destructive research. Live exploitation, phishing, credential attacks, denial of service, stealth, persistence, fraud, and third-party victim testing are outside scope.

Until a private disclosure address is published, report sensitive project vulnerabilities privately to the repository owner through GitHub's private vulnerability reporting feature when available.
